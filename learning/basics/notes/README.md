# notes — 本のまとめノート

『実用Go言語』「駆け足で学ぶGoの基本」章の内容を、**自分の言葉で書き直したノート**を置くディレクトリ。

## ここに置くもの

- 本の節を読んで、要点を自分の言葉でまとめたもの
- 用語の定義・コードの読み方・「なぜそう設計されているか」の理解
- 教科書的な「整った知識」を保存する場所

## ここに置かないもの（混同しがちなので注意）

| 内容 | 置く場所 |
|---|---|
| 「あれ？これどう動くんだっけ」という疑問・調査・解決の時系列 | `../scrap.md` |
| 実際に書いて動かしたGoコード | `../examples/` |
| ループの計画・進捗管理 | `../plan.md` |

## ファイル名規則

`<ループ番号>_<トピック名>.md` で統一する。

例:
- `01_hello.md` — Loop 1（Hello World + リテラル + 変数宣言 + 名前 + コメント）
- `02_types.md` — Loop 2（型と変換 + ゼロ値）
- `03_pointer.md` — Loop 3（ポインター）
- ...

## 書き方のコツ（ファインマン・テクニック）

1. 本の節を **一度読み切る**
2. 本を閉じる
3. **本を見ずに** 思い出した内容をこのファイルに書く
4. 書けなかった部分があれば本を開いて補う（書けなかった箇所こそ理解が浅い証拠）
5. 最後に、コード例や図解を加えて完成

「他人にGoを教えるならどう説明するか」を意識して書くと、深く定着する。
