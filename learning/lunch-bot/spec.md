# Slack ランチシャッフル bot — 要件定義

## 1. 概要

ターミナルで `go run ./cmd/bot` を叩くと「ランチ参加者募集」の投稿が出て、
🍱 リアクションを押した人を翌朝 09:00 にランダムで 3〜5 人組のグループに分け、
同じチャンネルに発表する bot。

> **Phase 1 設計判断**: 当初 `/lunch` スラッシュコマンドを検討したが、Socket Mode/WebSocket/slack-go ライブラリ導入が初学者には負担が大きいため、ターミナル起動方式に変更。Cloud Run 移行時にHTTPトリガーへ自然に置き換え可能。

## 2. ユーザーストーリー

| 役割 | 行動 | ゴール |
|---|---|---|
| ランチ幹事（=ご主人様） | `go run ./cmd/bot` を叩く | 募集を簡単に開始したい |
| 参加希望メンバー | 投稿に 🍱 を押す | 参加表明を1クリックで済ませたい |
| 全員 | 翌朝 09:00 に発表を見る | 誰とランチに行くか把握したい |

## 3. 機能要件

### 3.1 起動コマンド
- 起動方法: ターミナルで `go run ./cmd/bot`
- 動作: 設定済みチャンネル (`SLACK_CHANNEL_ID`) に募集投稿を出す
- 起動後はプロセスが生き続け、翌日 09:00 まで待機 → 集計→発表 → 終了
- 投稿テキスト案:
  ```
  🍽️ 今週のランチ参加者募集！
  参加したい人は 🍱 を押してね
  締切: 翌日 09:00 / 3〜5人組でランダムに振り分けます
  ```

### 3.2 リアクション集計
- 集計対象: 募集投稿に押された 🍱 (`bento`) リアクション
- 除外: bot 自身の場合（人間ユーザーのみ）
- 取得方法: Slack API `reactions.get`

### 3.3 締切と発表
- 締切時刻: 募集投稿の **翌日 09:00 (JST)**
- 動作: 翌日 09:00 に bot が自動で集計→グループ分け→発表
- スケジューリング: in-process（goroutine + `time.AfterFunc` を想定）

### 3.4 グループ分けアルゴリズム
- 基本: 4人組
- 柔軟サイズ: 3〜5 人で割り切れるよう調整
  - 3人 → [3]
  - 4人 → [4]
  - 5人 → [5]
  - 6人 → [3, 3]
  - 7人 → [3, 4] or [4, 3]
  - 8人 → [4, 4]
  - 9人 → [4, 5] or [3, 3, 3]
  - 10人 → [5, 5] or [4, 3, 3]
  - 11人 → [4, 4, 3] or [3, 4, 4]
  - 12人 → [4, 4, 4]
  - n人 → 4の倍数を基準に、3〜5 人組で構成
- メンバー選定: ランダムシャッフル（過去ペアは記録しない）

### 3.5 発表投稿
- 投稿先: 募集投稿と同じチャンネル
- フォーマット案:
  ```
  🎉 今週のランチグループ決定！

  グループA: <@U012> <@U345> <@U678> <@U901>
  グループB: <@U234> <@U567> <@U890>

  楽しいランチを！🍱
  ```
- メンション: `<@Uxxxx>` 形式で本人通知が飛ぶようにする

### 3.6 少人数時のフォールバック
- 参加者 0〜2 人の場合:
  ```
  😿 今週は参加者が少なかったのでお休みです
  また来週叩いてください
  ```

### 3.7 多重実行
- 同日に `go run ./cmd/bot` が複数回叩かれても**重複チェックしない**
- それぞれ独立した募集として動作（プロセスを別ターミナルで複数起動可能）

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| 実行環境 (Phase 1) | ローカル + Socket Mode で常時稼働 |
| 実行環境 (Phase 2) | Cloud Run + Cloud Scheduler |
| タイムゾーン | Asia/Tokyo (JST) |
| 言語 | Go 1.26 |
| アーキテクチャ | 3層レイヤード (handler / service / repository) |
| 永続化 | なし（in-memory のみ）。bot再起動で進行中の募集は失われる |
| 設定 | 環境変数 (`.env` から `source`) |

## 5. Slack App 設定

### 5.1 必要な OAuth Scope (Bot Token Scopes)
- `chat:write` — チャンネルに投稿 ✅ 既に設定済
- `reactions:read` — リアクションを取得 ✅ 既に設定済
- `users:read` — ユーザー情報取得（メンション用）✅ 既に設定済

### 5.2 Socket Mode
- **不要**（Phase 1 では使わない）

### 5.3 追加設定
- **不要**（既存の bot 招待済みチャンネルでそのまま動作）

## 8. 依存ライブラリ方針

- **標準ライブラリのみ** (`net/http`, `encoding/json`, `time`, `math/rand` など)
- サードパーティライブラリは **使わない**（Phase 1 ではゼロ依存で完結）

## 9. スコープ外（やらないこと）

- ❌ 過去ペア履歴の記録/回避
- ❌ 参加者の事前候補抽出（誰でも自由参加）
- ❌ ランチ場所の提案
- ❌ カレンダー連携
- ❌ 複数チャンネル対応
- ❌ Web UI
- ❌ DM 通知
- ❌ 締切の手動制御コマンド（`/lunch close` 等）
- ❌ Slack スラッシュコマンド対応（Phase 4 で検討）
- ❌ Socket Mode（Phase 4 で検討）
