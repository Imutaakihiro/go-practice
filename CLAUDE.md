# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリの目的

オライリー『実用Go言語』を読みながらGoを学ぶ**個人学習リポジトリ**。プロダクションコードではない。

ユーザーは初学者で、Goが本格的に学ぶ初の言語。これまでAI支援中心のいわゆる "vibeコーディング" で、エラーを自力で直す経験が浅い。**「自分で書ける・自分で直せる」状態になること**が裏のゴール。

## 学習ログの運用

`learning/<topic>/` をユニットに、トピック単位で学びを残す。

- `learning/<topic>/scrap.md` — Zennスクラップ風の学習ログ。`## YYYY-MM-DD HH:MM 見出し` で時系列に追記し、疑問→調査→解決の流れを残す
- `learning/<topic>/examples/` — そのトピックで書いた `.go` コード
- `learning/README.md` — トピック一覧表。新トピックを切ったらここに1行追加

新しい話題を始めるときの判断: 既存トピックに収まるなら追記、独立した概念なら新ディレクトリ。

## Go環境

- Go 1.26.2 (darwin/arm64)
- モジュールルート: リポジトリ直下の `go.mod` 1つ（`github.com/androots/go-practice`）
- 全 `examples/` 配下のコードはこの単一モジュールに属する

## よく使うコマンド

```bash
go run ./learning/<topic>/examples/<file>.go   # 単発実行（学習中の試し打ち）
go test ./learning/<topic>/...                 # パッケージ単位のテスト
gofmt -w .                                     # 整形（Goでは "正しい整形" は1つだけ）
go vet ./...                                   # 静的チェック（疑わしいコードを検出）
go mod tidy                                    # 依存関係を整理
```

## 教えるときの方針（重要）

ユーザーは "エラーを読み解く力" を育てている途中。実装/解説するときは:

- **暗記より仕組み**: 「なぜそう書くのか」「なぜこの記法なのか」を必ず添える
- **専門用語は使った瞬間に1行で説明**: 前提知識を当てにしない
- **エラーは一緒に読み解く**: `file:line:col: message` の構造から、英語の決まり文句の意味まで噛み砕く
- **抽象化を急がない**: インターフェイス・ジェネリクス・並行処理は必要になった時点で導入。最初から出さない
- **標準ライブラリ優先**: サードパーティ製ライブラリやフレームワークを安易に勧めない（まず素朴に書ける感覚を養う）
- 学んだ内容は対応する `scrap.md` に追記する流れで進める

## アーキテクチャ

学習用なので大袈裟な構造はない。`learning/<topic>/` は相互に独立しており、各トピックは「**本を読む → examples にコードを書く → scrap.md に理解を残す**」の3点セットで完結する設計。
